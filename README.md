# Pharmacy E2E Tests

Playwright E2E test suite for NHS pharmacy websites. Covers the full patient journey:

`/conditions` → condition detail → eligibility check → questionnaire → NHS PDS sign-up / Private booking → payment → thank you

Supports multiple pharmacy sites as separate Playwright **projects**, and includes a local **dashboard** for running tests, editing test data, and viewing results from a browser UI.

---

## Quick Start

```bash
npm install
npx playwright install chromium
npm run dashboard
# Opens at http://localhost:7890
```

---

## Running Tests

### npm scripts

| Command | Description |
|---|---|
| `npm test` | Run all tests headless |
| `npm run test:headed` | Run with a visible browser window |
| `npm run test:debug` | Step through each action interactively |
| `npm run test:ui` | Open Playwright UI mode |
| `npm run test:report` | View the last HTML report |

### Run against a specific pharmacy

```bash
npm run test:pharmaease     # Pharmaease
npm run test:paydens        # Paydens Pharmacy
npm run test:pharmacist     # The Pharmacist
npm run test:localhost      # http://localhost:4005
```

Or use `--project` directly:

```bash
npx playwright test --project="Pharmaease"
```

## Dashboard & Background Services

### The Dashboard

A local web dashboard lets you configure and run tests without touching the command line.

```bash
npm run dashboard
# http://localhost:7890
```

Features:
- **Select pharmacy** and **test file / individual test** to run
- **Live log stream** via SSE while tests are running
- **Edit test data** (user, payment, booking preferences) — changes are written back to `tests/fixtures/test-data.ts`
- **View videos and traces** generated by the last run
- **Launch Playwright UI** as a background service

### PM2 Services

Manage the dashboard and Playwright UI as background services using [PM2](https://pm2.keymetrics.io/).

| Command | Description |
|---|---|
| `npm run services:start` | Start dashboard (7890) and Playwright UI (8080) in background |
| `npm run services:status` | Check if services are running |
| `npm run services:logs` | View live logs for both services |
| `npm run services:stop` | Stop all services |

## Deployment

### 1. Oracle Cloud (Recommended for ARM64 Free Tier)

If you are running on an Oracle Cloud ARM64 VM (Ubuntu 22.04), use the automated setup script. This handles Swap, Node.js, PM2, system dependencies, and firewall configuration.

```bash
chmod +x deploy/oracle-setup.sh
./deploy/oracle-setup.sh
```

### 2. Docker (AMD64 / x86_64 only)

Run the dashboard and tests in a containerized environment.

```bash
docker-compose up -d
```
*Note: The Docker image is optimized for AMD64. For ARM64 (M1/M2 Mac or Oracle ARM), use the bare-metal setup via `oracle-setup.sh`.*

## GitHub Codespaces

### Dev Container

This repo includes a `.devcontainer` configuration. When opened in a Codespace, it automatically:
1. Installs all Node.js and Playwright dependencies.
2. Starts the Dashboard and Playwright UI services via PM2.
3. Forwards ports **7890** (Dashboard) and **8080** (Playwright UI) and makes them public.

### Keep-Alive (Cloudflare Worker)

To prevent your Codespace from shutting down during long-running test cycles or while you are away, use the Cloudflare Worker in the `keepalive/` folder.

See [keepalive/README.md](keepalive/README.md) for step-by-step setup instructions. It fires a GitHub Action every 10 minutes to ensure the Codespace remains active.

## Troubleshooting

**Tests time out on `/conditions`**
Ensure the target app is running and reachable. Check `BASE_URL` or the pharmacy `baseURL` in `pharmacies.ts`.

**NHS PDS check fails / no match**
Expected — the test user (John Smith, SW1A 1AA, 01/01/1990) is unlikely to match a real NHS record. The flow continues via the manual sign-up path.

**Videos and traces not found**
Artifacts are saved to `test-results/`. Run `npm run test:report` to open the HTML report, or browse artifacts from the dashboard after a run.
